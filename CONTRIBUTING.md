# راهنمای مشارکت

از مشارکت شما در توسعه فریمورک WeWork خوشحالیم! این راهنما به شما کمک می‌کند تا مشارکت موثری داشته باشید.

## نحوه مشارکت

### گزارش باگ

1. Issue جدید ایجاد کنید
2. توضیح کامل مشکل را بنویسید
3. مراحل بازتولید را ذکر کنید
4. نسخه فریمورک و Python را مشخص کنید

### پیشنهاد ویژگی جدید

1. Issue جدید با label "enhancement" ایجاد کنید
2. توضیح دهید چرا این ویژگی مفید است
3. مثال‌های استفاده را ارائه دهید

### ارسال Pull Request

1. Fork کنید
2. Branch جدید ایجاد کنید (`git checkout -b feature/amazing-feature`)
3. تغییرات را commit کنید (`git commit -m 'Add amazing feature'`)
4. Push کنید (`git push origin feature/amazing-feature`)
5. Pull Request ایجاد کنید

## استانداردهای کد

### Python

- از PEP 8 پیروی کنید
- از type hints استفاده کنید
- Docstrings برای توابع و کلاس‌ها
- تست‌ها را بنویسید

### JavaScript/React

- از ESLint پیروی کنید
- از JSDoc برای توضیحات استفاده کنید
- کامپوننت‌ها را functional بنویسید

## تست‌ها

قبل از ارسال PR، تست‌ها را اجرا کنید:

```bash
pytest tests/ -v
```

## مستندات

- مستندات را به‌روزرسانی کنید
- مثال‌های کد را اضافه کنید
- تغییرات را در CHANGELOG.md ذکر کنید

## سوالات

برای سوالات، از Issues استفاده کنید یا مستندات را مطالعه کنید.
